Текущая реализация метода Ляпунова некорректна для квантовой пены по следующим причинам:

## 1. Стохастическая природа системы

Квантовая пена использует `np.random` для создания частиц:
- Строка 157: `np.random.random() < prob` - решение о создании частицы
- Строка 175: `np.random.normal(0.0, 0.5)` - случайная масса
- Строки 469-472: `np.random.randint()` - случайные позиции

Когда мы делаем `copy.deepcopy(foam_simulator)`, мы копируем состояние системы, 
но НЕ копируем состояние генератора случайных чисел!

## 2. Что происходит на самом деле

```python
foam_original = QuantumFoam(...)
foam_perturbed = copy.deepcopy(foam_original)

# Добавляем малое возмущение к позиции одной частицы
foam_perturbed.particles[0].position[1] += 1e-8

# Эволюционируем обе системы
foam_original.evolve_foam(...)   # Создает частицы в случайных местах A
foam_perturbed.evolve_foam(...)  # Создает частицы в случайных местах B (РАЗНЫХ!)
```

Обе системы используют ОДИН И ТОТ ЖЕ глобальный `np.random`, поэтому:
- Первая система вызывает `np.random.random()` и получает значение X
- Вторая система вызывает `np.random.random()` и получает значение Y (следующее в последовательности)
- Частицы создаются в РАЗНЫХ местах!

## 3. Почему это проблема

Показатель Ляпунова измеряет чувствительность к НАЧАЛЬНЫМ УСЛОВИЯМ, а не к случайным флуктуациям.

Правильно:
- Две траектории с одинаковыми случайными событиями, но разными начальными условиями
- Расхождение из-за детерминированной динамики

Неправильно (текущая реализация):
- Две траектории с РАЗНЫМИ случайными событиями
- Расхождение из-за разных случайных чисел, а не из-за начального возмущения

## 4. Почему lambda ~ 40 так высоко

Высокие значения объясняются тем, что мы измеряем не чувствительность к начальным условиям,
а расхождение из-за РАЗНЫХ случайных событий в двух системах.

Когда в одной системе создается частица в точке A, а в другой - в точке B,
это создает огромное расхождение, не связанное с начальным возмущением.
