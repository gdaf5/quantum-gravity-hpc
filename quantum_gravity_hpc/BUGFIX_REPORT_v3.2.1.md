### 🔴 БАГ #1: СКАЛЯР КРЕЧМАНА - НЕПРАВИЛЬНЫЕ ИНДЕКСЫ

**Проблема:**
```python
# БЫЛО (НЕПРАВИЛЬНО):
K = torch.einsum('bijkl,bijkl->b', Riemann, Riemann)
```
Вычислял R^ρ_σμν × R^ρ_σμν вместо R^αβγδ × R_αβγδ

**Исправление:**
```python
# СТАЛО (ПРАВИЛЬНО):
def compute_kretschmann_scalar(self, Riemann, g, g_inv):
    # 1. Опустить первый индекс
    Riemann_lower = torch.einsum('bai,bijkl->bajkl', g, Riemann)
    
    # 2. Поднять все индексы
    Riemann_upper = torch.einsum('bas,bjt,bku,blv,bstuv->bijkl', 
                                  g_inv, g_inv, g_inv, g_inv, Riemann_lower)
    
    # 3. Корректная свертка
    K = torch.einsum('bijkl,bijkl->b', Riemann_upper, Riemann_lower)
    return K
```

**Файл**: `ml_metric_predictor.py:210-237`  
**Критичность**: 🔴 ВЫСОКАЯ - неправильный физический инвариант  
**Статус**: ✅ ИСПРАВЛЕНО

### 🔴 БАГ #2: ВТОРОЕ ТОЖДЕСТВО БИАНКИ - ОБЫЧНАЯ ПРОИЗВОДНАЯ

**Проблема:**
Использовал обычную производную ∂_ν G^μν вместо ковариантной ∇_ν G^μν

**Исправление:**
Добавлен новый метод `compute_covariant_divergence_einstein`:
```python
def compute_covariant_divergence_einstein(self, G, Gamma, dG):
    """
    ∇_ν G^μν = ∂_ν G^μν + Γ^μ_νλ G^λν + Γ^ν_νλ G^μλ
    """
    covariant_div = torch.zeros(batch_size, 4, ...)
    
    for mu in range(4):
        for nu in range(4):
            # Обычная производная
            covariant_div[:, mu] += dG[:, nu, mu, nu]
            
            # Члены с Кристоффелем
            for lam in range(4):
                covariant_div[:, mu] += Gamma[:, mu, nu, lam] * G[:, lam, nu]
                covariant_div[:, mu] += Gamma[:, nu, nu, lam] * G[:, mu, lam]
    
    return covariant_div
```

Обновлен метод `verify_bianchi_identities` для использования ковариантной производной.

```python
# СТАЛО (БЫСТРО):
def compute_riemann_tensor(self, Gamma, dGamma):
    # Производные через permute
    R_deriv = dGamma.permute(0, 2, 4, 1, 3) - dGamma.permute(0, 2, 4, 3, 1)
    
    # Квадратичные члены через einsum
    R_quad1 = torch.einsum('brml,blns->brsmn', Gamma, Gamma)
    R_quad2 = torch.einsum('brnl,blms->brsnm', Gamma, Gamma)
    
    Riemann = R_deriv + R_quad1 - R_quad2
    return Riemann
```

## 📚 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Правильное вычисление скаляра Кречмана:

**Математика:**
```
K = R^{αβγδ} R_{αβγδ}

Шаг 1: R_{ασμν} = g_{αρ} R^ρ_{σμν}
Шаг 2: R^{αβγδ} = g^{αs} g^{βt} g^{γu} g^{δv} R_{stuv}
Шаг 3: K = Σ R^{αβγδ} R_{αβγδ}
```

**Код:**
```python
Riemann_lower = torch.einsum('bai,bijkl->bajkl', g, Riemann)
Riemann_upper = torch.einsum('bas,bjt,bku,blv,bstuv->bijkl', 
                              g_inv, g_inv, g_inv, g_inv, Riemann_lower)
K = torch.einsum('bijkl,bijkl->b', Riemann_upper, Riemann_lower)
```

### Ковариантная производная Эйнштейна:

**Математика:**
```
∇_ν G^μν = ∂_ν G^μν + Γ^μ_νλ G^λν + Γ^ν_νλ G^μλ
```

**Физический смысл:**
В искривленном пространстве обычная производная не сохраняет тензорный характер. Нужно добавить члены с символами Кристоффеля, чтобы учесть кривизну.

### Оптимизация через einsum:

**Преимущества:**
- Векторизация на GPU
- Автоматическая оптимизация PyTorch
- Эффективный backprop
- Читаемый код

**Пример:**
```python
# Вместо 5 циклов (1024 итерации):
for rho, sigma, mu, nu, lam in itertools.product(range(4), repeat=5):
    R[rho,sigma,mu,nu] += Gamma[rho,mu,lam] * Gamma[lam,nu,sigma]

# Одна строка:
R = torch.einsum('brml,blns->brsmn', Gamma, Gamma)
```
