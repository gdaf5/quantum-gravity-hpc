# Руководство по запуску экспериментов для диссертации

## Быстрый старт

### 1. Проверка установки
```bash
python --version  # Должен быть Python 3.8+
pip list | grep -E "torch|numpy|h5py|matplotlib|scipy"
```

### 2. Быстрый тест (5 минут)
```bash
python run_all_experiments.py --quick
```

Результат:
- `dissertation_results/full_report.json` - полный отчет
- `dissertation_results/summary.txt` - текстовое резюме
- `analysis_results/*.png` - графики

### 3. Создание PDF презентации
```bash
python create_dissertation_pdf.py
```

Результат: `dissertation_visualization.pdf` - готовый отчет для защиты

## Интерпретация результатов

### Ключевые метрики

#### 1. Фрактальная размерность (D2)
Файл: `analysis_results/fractal_dimension.png`

**Интерпретация:**
- **D2 < 3.0**: Коллапс в нижнюю размерность (сильная гравитация)
- **D2 ≈ 3.0**: Классическое поведение (слабые квантовые эффекты)
- **D2 > 3.0**: Нелокальные квантовые эффекты (активная квантовая пена)
- **D2 >> 3.0**: Доминирование квантовых флуктуаций

**Для диссертации:**
```
Если D2 > 3.0, то можно утверждать:
"Обнаружены нелокальные квантовые эффекты на планковском масштабе,
превышающие классические предсказания на (D2 - 3.0) × 100%"
```

#### 2. Показатель Ляпунова (λ)
Файл: `analysis_results/lyapunov_exponent.png`

**Интерпретация:**
- **λ > 0**: Хаотическая динамика (экспоненциальное расхождение траекторий)
- **λ ≈ 0**: Регулярное движение
- **λ < 0**: Сходящиеся траектории (аттрактор)

**Для диссертации:**
```
Положительный показатель Ляпунова λ = X.XXXX указывает на
детерминированный хаос в квантовой пене с характерным временем
расхождения τ = 1/λ планковских времен.
```

#### 3. Дисперсия скоростей (квантовая диффузия)
Файл: `analysis_results/velocity_dispersion.png`

**Интерпретация:**
```
D_quantum = σ²(v_final) / σ²(v_initial)
```

- **D_quantum > 1**: Рост дисперсии (квантовая диффузия)
- **D_quantum = 1**: Сохранение дисперсии
- **D_quantum < 1**: Уменьшение дисперсии (фокусировка)

**Для диссертации:**
```
Коэффициент квантовой диффузии D = X.XX показывает увеличение
дисперсии импульса на XX% за время симуляции, что соответствует
эффективной температуре квантовой пены T_eff = ...
```

#### 4. Дрейф энергии
Файл: `analysis_results/energy_conservation.png`

**Интерпретация:**
- **< 1%**: Отличная численная стабильность
- **1-5%**: Приемлемая точность
- **> 5%**: Требуется уменьшение шага интегрирования

**Для диссертации:**
```
Относительный дрейф энергии составил X.XX%, что подтверждает
численную стабильность симплектического интегратора на
планковском масштабе.
```

#### 5. Эффект обратного влияния
Файл: `backreaction_comparison.png`

**Интерпретация:**
```
Δ = R_with_backreaction - R_without_backreaction
```

- **Δ > 0**: Обратное влияние усиливает расширение
- **Δ < 0**: Обратное влияние вызывает коллапс
- **Δ ≈ 0**: Слабое обратное влияние

**Для диссертации:**
```
Учет обратного влияния материи на метрику приводит к изменению
радиуса кластера на ΔR = X.XX планковских длин, что составляет
XX% от классического значения.
```

## Типичные проблемы и решения

### Проблема 1: Недостаточно памяти
```
MemoryError или RuntimeError: out of memory
```

**Решение:**
- Уменьшите `n_particles` (например, с 1000 до 500)
- Уменьшите `grid_shape` (например, с 8×8×8×8 до 6×6×6×6)
- Уменьшите `n_steps` (например, с 100 до 50)

### Проблема 2: Слишком долго выполняется
```
Симуляция работает > 30 минут
```

**Решение:**
- Используйте GPU (если доступен):
  ```python
  device = 'cuda' if torch.cuda.is_available() else 'cpu'
  ```
- Уменьшите количество шагов
- Используйте `--quick` режим для тестирования

### Проблема 3: Большой дрейф энергии (> 5%)
```
energy_drift_percent > 5.0
```

**Решение:**
- Уменьшите шаг интегрирования `dt` в `engine.py`:
  ```python
  dt = 1e-45  # уменьшите до 5e-46 или 1e-46
  ```
- Используйте RK4 вместо Verlet (медленнее, но точнее)

### Проблема 4: NaN или Inf в результатах
```
RuntimeError: Found NaN or Inf
```

**Решение:**
- Уменьшите амплитуду флуктуаций метрики
- Добавьте ограничение на метрику (clamp):
  ```python
  self.metric_grid[..., 0, 0] = torch.clamp(self.metric_grid[..., 0, 0], -2.0, -0.5)
  ```
